You are evaluating a single prior authorization criterion against a patient's
clinical evidence.

Criterion (verbatim from policy):
{criterion_text}

Criterion type: {criterion_type}

Patient evidence (one entry per line; use the id when citing):
{evidence_block}

Decide whether the criterion is met for this patient.

Output a single JSON object with:
- status ("met" | "not_met" | "partial" | "insufficient_evidence")
- supporting_evidence_ids (array of evidence ids, may be empty)
- reasoning (one or two sentences, no quoted criterion text)
- confidence (float in [0, 1] reflecting your certainty in the status call)

Rules:
- Use "insufficient_evidence" when the chart simply does not address the
  criterion. Use "not_met" only when evidence affirmatively contradicts it.

- For criterion_type == "required": evaluate whether the clinical situation
  matches. Cite chart evidence that demonstrates the match.

- For criterion_type == "contraindication": "met" means the contraindication
  IS present (the patient should be denied). "not_met" means the
  contraindication is absent (good for approval). Absence of mention in the
  chart should usually be "not_met" rather than "insufficient_evidence",
  because contraindications are typically actively documented when present.

- For criterion_type == "documentation": these describe what the requester
  must submit with the prior auth packet, not facts that must already be
  in the chart. Mark "met" whenever the chart contains the underlying
  clinical fact the document would attest to. Mark "not_met" or
  "insufficient_evidence" only if that underlying clinical fact is itself
  missing from the chart.

- Cite only evidence ids that actually appear in the list above.

Return only valid JSON. No prose, no code fence.
